# CLAUDE.md

このファイルは、このリポジトリでコードを扱う際の Claude Code (claude.ai/code) 向けガイダンスです。

## プロジェクト概要

- **プロジェクト名**: task-board
- **内容**: タスク管理（カンバンボード）アプリ。「ToDo / 進行中 / 完了」などの列にタスクカードを並べ、追加・移動・削除しながら作業状況を管理できる Web アプリケーション。
- **技術スタック**: HTML / CSS / JavaScript（フレームワークなしのバニラ構成）

## Git 運用ルール

- **コードを変更するたびに、必ず GitHub へプッシュする。**
  - 変更が一区切りついたら `git add` → `git commit` → `git push` を実施する。
  - コミットメッセージは変更内容が分かるように日本語で簡潔に記述する。
  - 作業を未コミット・未プッシュのまま放置しない。
- リモートリポジトリを基本（single source of truth）とし、ローカルの変更は速やかに反映する。

```bash
# 変更を反映する基本フロー
git add .
git commit -m "変更内容を簡潔に記述"
git push
```

## 開発環境

- ビルドツールやパッケージマネージャは使用しない素の HTML/CSS/JS 構成。
- 実行はブラウザで `index.html` を直接開くか、簡易サーバーで配信する。

```bash
# 簡易ローカルサーバーで起動する場合（Python が利用可能なとき）
python -m http.server 8000
# → ブラウザで http://localhost:8000 を開く
```

## 想定するファイル構成

```
task-board/
├── index.html      # アプリのエントリポイント（マークアップ）
├── css/
│   └── style.css   # スタイル定義
└── js/
    └── app.js      # タスク管理のロジック（追加・移動・削除・保存）
```

※ 上記は推奨構成。実際のファイル追加時はこの方針に沿って配置する。

## 設計方針・コーディング規約

- **言語**: ユーザーへの返答・コメント・UI 文言は日本語を基本とする。
- **HTML**: セマンティックな要素を使い、構造とスタイル・ロジックを分離する。
- **CSS**: スタイルは CSS ファイルに集約し、インラインスタイルは避ける。
- **JavaScript**:
  - タスクのデータ（タイトル・状態など）はデータ構造（配列・オブジェクト）として分離し、ロジックから切り離す。
  - DOM 操作・状態管理・保存処理は責務ごとに関数へ分割する。
  - グローバル変数の汚染を避ける。
- 外部ライブラリやフレームワークの導入は原則行わない。必要になった場合は事前に方針を確認する。

## 主要機能（想定）

- タスクの追加
- タスクの状態変更（列間の移動：ToDo → 進行中 → 完了）
- タスクの削除
- タスクの永続化（localStorage など、必要に応じて）
